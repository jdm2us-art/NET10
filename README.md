# ОТВЕТ:

 В каких пакетах передается AreaID?

AreaID передается ВО ВСЕХ типах OSPF пакетов:

 Hello (Type 1)

 Database Description (Type 2)

 Link State Request (Type 3)

 Link State Update (Type 4)

 Link State Acknowledgment (Type 5)

AreaID находится в OSPF заголовке, который является общим для всех типов пакетов.

 В каком формате передается AreaID?

AreaID передается в 32-битном формате:

В десятичном формате с точками (для отображения):

Например: 0.0.0.0, 0.0.0.1, 1.2.3.4

В шестнадцатеричном формате (в сырых данных пакета):

Например: 0x00000000, 0x00000001, 0x01020304

В двоичном формате (для передачи по сети):

4 байта = 32 бита

Например: 00000000 00000000 00000000 00000000

Приведите ответы на вопросы в свободной форме.

# ОТВЕТ: 

Через LSA (Link State Advertisement) в пакетах LSU (Link State Update)

Маршрутизатор (Router4) обнаруживает изменение состояния интерфейса (Loopback 0 стал down)

Генерирует новую LSA (Link State Advertisement) типа 1 (Router LSA) с:

Увеличенным Sequence Number (например, с 0x80000004 до 0x80000005)

Обновленным списком линков (без Loopback 0)

Уменьшенным количеством линков

Помещает LSA в пакет LSU (Link State Update) и отправляет:

Назначение: 224.0.0.6 (AllDRouters) - только DR и BDR

Причина: DROTHER отправляют обновления только DR и BDR

DR (Router1) получает LSU, обновляет свою LSDB и:

Отправляет LSAck (подтверждение) обратно Router4

Распространяет обновление всем маршрутизаторам через 224.0.0.5 (AllSPFRouters)

Все маршрутизаторы получают обновление:

Обновляют свои LSDB (Link State Database)

Запускают алгоритм SPF (Shortest Path First) для пересчета маршрутов

Удаляют из таблицы маршрутизации сеть 192.168.40.0/24

# Задание 4. Определение DR и BDR

Сделайте предположение, какие маршрутизаторы будут выбраны в качестве DR и BDR. Узнайте, какие маршрутизаторы выбраны DR и BDR в реальности при помощи команд на маршрутизаторе. Какую команду необходимо использовать для получения информации о DR/BDR? Сошлись ли предположения с реальностью? Если нет, то почему?

Приведите ответы на вопросы в свободной форме.

# ОТВЕТ: 

 Команды для получения информации о DR/BDR:
 Основная команда:

show ip ospf interface

 Для конкретного интерфейса:

show ip ospf interface FastEthernet0/0

 Сокращенный вариант:

show ip ospf int fa0/0

 Просмотр соседей с указанием ролей:

show ip ospf neighbor

Предположения НЕ совпали с реальностью из-за порядка запуска маршрутизаторов

DR = Router1 (1.1.1.1), а не Router4 с большим Router ID

BDR = Router2 (2.2.2.2), а не Router3

DR/BDR не переизбираются при появлении новых маршрутизаторов

В OSPF выбор DR и BDR зависит НЕ ТОЛЬКО от приоритетов и Router ID,
но и от ПОРЯДКА ЗАПУСКА маршрутизаторов!
